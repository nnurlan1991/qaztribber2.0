# План переориентации QazTriber в локальное веб-приложение

## 1. Цель и границы первой версии

QazTriber становится локальным веб-приложением: интерфейс открывается в браузере на Mac, а распознавание выполняется локальным Python-сервером. Интернет нужен только для первоначальной установки зависимостей и загрузки весов моделей. Публикация сервиса в интернет, учётные записи и многопользовательский режим в первую версию не входят.

Модели распознавания строго ограничены двумя вариантами GigaAM Multilingual:

| Отображение в интерфейсе | Идентификатор GigaAM | Назначение |
| --- | --- | --- |
| Быстрая — 220M | `multilingual_ctc` | Черновая и повседневная расшифровка |
| Точная — 600M | `multilingual_large_ctc` | Максимальное качество для казахского и русского |

> В репозитории GigaAM модель меньшего размера называется 220M, а не 200M. В интерфейсе и документации приложения использовать точное значение 220M.

## 2. Целевая локальная архитектура

```text
Браузер (http://127.0.0.1:5173)
  React + Vite + TypeScript
  ├─ запись с микрофона
  ├─ выбор и обрезка аудио
  ├─ выбор 220M / 600M
  └─ просмотр, копирование и сохранение текста
             │ HTTP + Server-Sent Events (прогресс)
             ▼
Локальный API (http://127.0.0.1:8000)
  FastAPI + Python 3.11
  ├─ приём и валидация файлов
  ├─ очередь задач (одна задача на GPU/MPS)
  ├─ менеджер моделей GigaAM
  └─ хранение временных файлов и результатов
             │
             ▼
Apple Silicon: PyTorch MPS → GigaAM
fallback: CPU
```

Работа через `localhost` не требует облачного сервера: аудиофайлы не покидают Mac. Одновременно в памяти держится только выбранная модель; при смене 220M/600M предыдущая выгружается. Это важно для стабильной работы на общей памяти Apple Silicon.

## 3. Новая структура репозитория

```text
backend/
  app/
    main.py                 # FastAPI, CORS, запуск API
    config.py               # пути, лимиты, локальные настройки
    api/transcriptions.py   # HTTP-эндпоинты задач
    services/gigaam.py      # загрузка, выгрузка и инференс GigaAM
    services/jobs.py        # состояния задач, отмена, прогресс
    services/audio.py       # ffmpeg, длительность, нормализация, фрагменты
    schemas.py              # Pydantic-схемы API
  tests/
  requirements.txt
frontend/
  src/
    app/                    # корневой экран и состояние
    features/transcription/ # загрузка, выбор модели, результат
    components/             # повторно используемые UI-компоненты
    lib/api.ts              # клиент API и SSE
  package.json
  vite.config.ts
README_RU.md
IMPLEMENTATION_PLAN_RU.md
```

Папка `models/` внутри репозитория не используется для весов. Кэш моделей и временные задания хранятся вне исходников, например в `~/Library/Application Support/QazTriber/`. В репозиторий они не попадут.

## 4. Backend: FastAPI и GigaAM

### 4.1 Менеджер моделей

Создать `GigaAMService` с единственной публичной операцией распознавания.

- При первом выборе модели вызвать `gigaam.load_model()` с её фиксированным идентификатором.
- Установить кэш Hugging Face/GigaAM в каталог данных приложения до загрузки модели.
- Определять устройство: `mps`, если оно доступно, иначе `cpu`.
- Не принимать путь к весам, произвольный Hugging Face ID или Whisper-модель из API.
- Перед загрузкой другой модели выгружать текущую, очищать память PyTorch и собирать мусор.
- Передавать в UI понятные стадии: «скачивание», «загрузка в память», «распознавание», «готово» и «ошибка».

### 4.2 Длинные аудио

У GigaAM прямой метод `transcribe` предназначен для аудио до 25 секунд. Поэтому обработка должна быть двухуровневой:

1. Для коротких файлов вызывать прямое распознавание.
2. Для длинных — в первой версии делить нормализованное аудио на фрагменты до 20 секунд с небольшим перекрытием и объединять текст без дублирования на границах.
3. Подготовить отдельный адаптер `LongformTranscriber` для официального `transcribe_longform`. Он включается только при установленных дополнениях GigaAM longform и заданном Hugging Face token для `pyannote/segmentation-3.0`.

Так приложение остаётся рабочим офлайн после загрузки весов, а улучшенный VAD-режим можно подключить позднее без переделки API.

### 4.3 Очередь и временные данные

- На первом этапе использовать внутреннюю очередь процесса и разрешать одну активную задачу: MPS не будет перегружен двумя 600M-инференсами одновременно.
- Каждой задаче присваивать UUID и состояние `queued`, `preparing`, `loading_model`, `transcribing`, `completed`, `failed` или `cancelled`.
- Хранить исходник, WAV после нормализации и результат во временной папке задачи; удалять их по кнопке «Очистить» и при старте приложения удалять старые задания.
- Поддержать отмену: между фрагментами проверять флаг задачи и прекращать дальнейшую обработку.

### 4.4 API первой версии

| Метод | Маршрут | Назначение |
| --- | --- | --- |
| `GET` | `/api/health` | проверка, что локальный сервер запущен |
| `GET` | `/api/models` | два доступных варианта и их статус в кэше |
| `POST` | `/api/transcriptions` | загрузка аудио и создание задачи; поле `model` принимает только `220m` или `600m` |
| `GET` | `/api/transcriptions/{id}` | текущий статус, процент и ошибка |
| `GET` | `/api/transcriptions/{id}/events` | SSE-поток прогресса |
| `POST` | `/api/transcriptions/{id}/cancel` | отмена задачи |
| `GET` | `/api/transcriptions/{id}/result` | текст и метаданные результата |
| `GET` | `/api/transcriptions/{id}/result.txt` | загрузка TXT-файла |
| `DELETE` | `/api/transcriptions/{id}` | удаление временных файлов задачи |

Проверки API: допустимые форматы WAV/MP3/M4A/FLAC/OGG, ограничение размера, безопасное имя файла, отказ от путей пользователя и понятная ошибка, если `ffmpeg` не найден.

## 5. Frontend: браузерный интерфейс

### 5.1 Функциональные экраны

Один рабочий экран без лишней навигации:

1. **Выбор аудио** — drag-and-drop, системный выбор файла и запись с микрофона через `MediaRecorder`.
2. **Подготовка** — название, длительность, аудиоплеер и волна; обрезка начала/конца перед отправкой.
3. **Модель** — две крупные карточки GigaAM с точным размером, назначением и статусом «не загружена / доступна локально».
4. **Выполнение** — последовательный прогресс: отправка, загрузка модели, фрагмент `N/M`, завершение; кнопка отмены.
5. **Результат** — редактируемый текст, копирование, скачивание TXT, запуск новой задачи и очистка локальной задачи.

### 5.2 Визуальное направление

Интерфейс — тёмная «аудиостудия»: глубокий графитовый фон, спокойные янтарные акценты для активного состояния, контрастная типографика и информативная звуковая волна. Это не копия desktop-интерфейса: управление строится вокруг последовательности «аудио → модель → текст», адаптируется к узкому окну браузера и доступно с клавиатуры.

### 5.3 Технические требования интерфейса

- React, TypeScript и Vite; без серверного рендеринга, так как приложение локальное.
- `wavesurfer.js` или эквивалент для волны и границ обрезки.
- Запись через браузерный `MediaRecorder`; полученный Blob отправляется как обычный файл.
- API-адрес через переменную окружения Vite; в разработке настроить proxy к FastAPI.
- SSE использовать для статусов, а HTTP polling оставить резервом при потере соединения.
- Не хранить записи и результаты в `localStorage` без явного действия пользователя.

## 6. Что будет удалено или заменено

Удаление выполнить только после запуска и проверки веб-версии.

- `main.py` — desktop-окно, работа с Tk, запись и аудиоредактор на Python.
- `engine/whisper_engine.py` — OpenAI Whisper и Transformers.
- `engine/espnet_engine.py` — ESPnet и обход `typeguard`.
- `engine/base.py` — заменить API-сервисом либо сохранить только если он полезен в новом backend-слое.
- `build_mac.sh` и упоминания PyInstaller/macOS `.app`.
- старые пользовательские параметры: файл `.pt`, произвольный Hugging Face repo и перечень Whisper-размеров.
- документация, описывающая desktop-поведение и старые зависимости.

## 7. Зависимости и запуск на Mac

### Системные зависимости

- macOS 14+ на Apple Silicon желательно;
- Python 3.11;
- Node.js 20 LTS;
- FFmpeg (`brew install ffmpeg`).

### Python-пакеты

Базовый набор: `fastapi`, `uvicorn`, `python-multipart`, `pydantic`, `gigaam`, `torch`, `torchaudio`, `soundfile`, `numpy`. Для долгого официального режима — дополнительный набор GigaAM longform и Hugging Face token.

### Команды разработчика

В README добавить отдельные команды для:

```bash
# Терминал 1
cd backend
python -m uvicorn app.main:app --reload --port 8000

# Терминал 2
cd frontend
npm run dev
```

Позже добавить одну команду `./run_local.sh`, которая проверяет `ffmpeg`, запускает backend и открывает браузер.

## 8. Порядок реализации

1. Создать backend-каркас, конфигурацию путей и `/api/health`.
2. Интегрировать GigaAM 220M, короткий WAV и один синхронный тест распознавания на MPS/CPU.
3. Реализовать менеджер модели, статусы, временную задачу и ограничение одной очереди.
4. Добавить загрузку форматов через FFmpeg, нормализацию в 16 kHz mono и обработку длинного аудио фрагментами.
5. Создать React-интерфейс загрузки, выбора 220M/600M и просмотра прогресса.
6. Добавить запись из браузера, волну и обрезку.
7. Подключить результат, копирование, TXT-экспорт, отмену и очистку.
8. Протестировать 600M и исправить поведение памяти на MPS.
9. Удалить desktop-код и переписать README/архитектурную документацию.
10. Добавить `run_local.sh` и инструкцию диагностики типовых ошибок.

## 9. Проверка готовности

Веб-версия считается готовой к личному использованию, если:

- приложение открывается локально в браузере и не требует облачного аккаунта;
- доступны только GigaAM 220M и GigaAM 600M;
- обе модели загружаются, отображают ход загрузки и освобождают память при смене;
- распознаются WAV, MP3, M4A, FLAC и OGG на казахском, русском и смешанной речи;
- запись с микрофона браузера проходит весь путь до результата;
- длинная запись не блокирует интерфейс, показывает прогресс, может быть отменена;
- результат можно копировать и скачать в TXT;
- после закрытия задачи её временные аудиоданные удаляются;
- в проекте не остаётся рабочего пути запуска Whisper, ESPnet или desktop-приложения.

## 10. Риски и решения

| Риск | Решение |
| --- | --- |
| 600M не помещается комфортно в память или работает медленно | загружать одну модель, оставить 220M вариантом по умолчанию, измерить время и память на конкретном Mac |
| GigaAM longform требует доступ к Hugging Face | базовый режим фрагментов работает без этого; longform подключается явно как опция |
| Браузерный микрофон не работает без разрешения | показать понятное состояние и инструкцию по разрешению macOS/браузера |
| FFmpeg отсутствует | проверять при старте и показать точную команду установки |
| Одновременные операции перегружают MPS | очередь из одной активной задачи и кнопка отмены |
