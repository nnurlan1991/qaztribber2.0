# Архитектура QazTriber Web

## Локальная модель работы

```text
Browser → React/Vite → FastAPI (127.0.0.1) → ИИ модель → MPS / CPU
```

Сервис слушает только loopback-интерфейс `127.0.0.1`. Пользовательский аудиофайл не отправляется на внешний сервер. Интернет используется только при первом скачивании зависимостей и весов ИИ модели.

## Frontend

`frontend/` — React/Vite-приложение. Оно даёт пять действий: выбрать или записать аудио, обозначить границы обрезки, выбрать 220M/600M, наблюдать прогресс и получить текст/TXT.

Frontend не знает путей к весам и не может передать произвольный Hugging Face ID. Он работает только с локальным API `/api`.

## Backend

`backend/app/main.py` создаёт FastAPI-приложение и локальные каталоги данных. `api/transcriptions.py` принимает файлы и публикует SSE-поток статуса.

`JobManager` имеет одну рабочую очередь. Это намеренно: два параллельных инференса 600M на MPS могут истощить общую память Mac. Задача движется через `queued`, `preparing`, `loading_model`, `transcribing` к `completed`, `failed` или `cancelled`.

## ИИ модель для распознавания

`GigaAMService` содержит разрешённый список:

- `220m` → `multilingual_ctc`;
- `600m` → `multilingual_large_ctc`.

Он хранит в памяти только один экземпляр модели, перед переключением освобождает память PyTorch и автоматически выбирает `mps` либо `cpu`. Веса кэшируются в `~/Library/Application Support/QazTriber/models`.

`ModelPreloadManager` последовательно загружает 220M и 600M по явной команде пользователя. Это сохраняет оба чекпойнта на диск для последующей офлайн-работы, но выгружает модель из памяти по окончании.

## Языки и темы

Интерфейс использует единую светлую премиальную тему. В задаче также сохраняется ожидаемый язык: `kazakh`, `russian` или `mixed`. Сама ИИ модель для распознавания не принимает флаг принудительного языка и автоматически распознаёт смешанную казахско-русскую речь; поэтому этот выбор честно обозначает ожидаемый язык записи, а не подменяет поведение модели.

## Аудио

`services/audio.py` через FFmpeg приводит входной файл к mono WAV 16 kHz. Короткие записи отправляются в ИИ модель напрямую. Длинные делятся на фрагменты до 20 секунд с небольшим перекрытием; текст на стыке очищается от повторяющихся слов.

## Данные

Каждой задаче присваивается UUID. Её исходник, нормализованный WAV, фрагменты и TXT лежат в `~/Library/Application Support/QazTriber/jobs/<uuid>/`. Пользователь может удалить законченную задачу из интерфейса. Эти данные не должны храниться в Git.
