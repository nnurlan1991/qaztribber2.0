# QazTriber — локальная веб-расшифровка KZ/RU для Windows

QazTriber устанавливается на Windows и открывается в браузере. Аудио обрабатывается локально: Python API загружает GigaAM в память только во время расшифровки, а браузер отвечает за выбор файла, запись и показ текста.

Доступны ровно две модели:

| В интерфейсе | GigaAM | Для чего |
| --- | --- | --- |
| Быстрая — 220M | `multilingual_ctc` | Обычные заметки и быстрые черновики |
| Точная — 600M | `multilingual_large_ctc` | Более требовательные записи и максимум качества |

> Меньшая официальная модель GigaAM имеет 220M параметров, не 200M.

## Что умеет

- Локальная расшифровка казахской, русской и смешанной речи.
- Загрузка WAV, MP3, M4A, FLAC, OGG и запись с микрофона в браузере.
- Обрезка начала и конца записи перед распознаванием.
- Светлый премиальный интерфейс аудиостудии для Windows и мобильного браузера.
- Выбор ожидаемого языка: казахский, русский или смешанный KZ + RU. GigaAM Multilingual автоматически определяет речь; смешанный режим — рекомендуемый для одного двуязычного файла.
- Кнопка «Скачать обе модели» для подготовки 220M и 600M к работе без интернета.
- Прогресс загрузки модели и транскрибации, отмена задачи.
- Редактирование, копирование и экспорт результата в TXT.
- Обработка длинных записей небольшими фрагментами без передачи аудио во внешний сервис.

## Требования

- Windows 10 или 11, 64-bit.
- Интернет только для первого скачивания весов моделей.
- Свободное место на диске для моделей и достаточно оперативной памяти для выбранной модели.

## Быстрый запуск

Скачайте `QazTriber-Windows-Setup.exe` из GitHub Releases и установите его. После запуска приложение само откроет браузер.

При первом запуске выбранной модели её веса будут скачаны в:

```text
%LOCALAPPDATA%\QazTriber\models
```

Можно не ждать первого распознавания: нажмите в интерфейсе «Скачать обе модели». Загрузка идёт последовательно и показывает количество скачанных мегабайт. После завершения можно отключить интернет; веса обеих моделей останутся на диске. Одновременно в памяти находится только активная модель, поэтому после закрытия приложения оперативная память освобождается. Установленные модели можно удалить в интерфейсе.

Временные аудиофайлы и результаты задач лежат рядом, в `jobs/`; они удаляются кнопкой «Новая запись» после завершения работы с результатом.

## Ручной запуск для разработки

```bash
# Терминал 1
cd backend
python3.11 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

```bash
# Терминал 2
cd frontend
npm install
npm run dev
```

## Проверки

```bash
backend/.venv/bin/python -m pytest backend/tests -q
cd frontend && npm run build
```

## Структура

- `backend/` — FastAPI, очередь задач, подготовка аудио и GigaAM.
- `frontend/` — React-интерфейс локальной аудиостудии.
- `IMPLEMENTATION_PLAN_RU.md` — детальный план и архитектурные решения.

## Установщик для Windows

GitHub Actions автоматически собирает запускаторы при публикации тега вида `v1.0.0`:

- `QazTriber-Windows-Setup.exe` — обычный Windows-установщик.

Приложение запускает браузерный интерфейс самостоятельно — Python, Node.js и FFmpeg друзьям не нужны. Веса моделей не входят в Git или установщик: внутри приложения нажмите «Скачать обе модели» один раз, после чего расшифровка работает офлайн. На Windows модели постоянно хранятся в `%LOCALAPPDATA%\\QazTriber\\models`; в интерфейсе видны путь, размер и кнопка удаления каждой модели.

Desktop-версия на `customtkinter`, Whisper и ESPnet удалена: в приложении нет загрузки произвольных моделей, `.pt`-файлов или облачного API.
